ATG Field Cheat Sheet

TLS-450PLUS + TLS-350 • quick workflows, common alarms, PLLD checks, and site mapping.

Common Alarms PLLD / Line Tests Site Mapping
Best practice: Screenshot first → verify physically → document what changed → recheck status. Menu labels vary by software/options; don’t change programming in the field unless authorized.
Workflow

60-Second Triage (use every call)

Do this first

  1. Active vs. history (Are you responding to something happening now?)
  2. What object? Tank # / Line # / Sensor ID / Input
  3. Screenshot evidence (alarm + details)
  4. Classify: Containment/Sensor • Line/PLLD • Tank • System/Setup

Document standard

  • Site name + date/time
  • Tank/Line/Sensor ID (exact)
  • What you saw (symptom)
  • What you checked physically
  • What you fixed/changed
  • Recheck result (normal? still alarm?)
Quick Reference

Common Alarms (meaning + first moves)

These are the ones you listed as most common in your world. Keep it simple: verify + fix + document.

Fuel Alarm (most common)

Meaning: Fuel (liquid) is present in an area being monitored by a sensor — i.e., a space it’s not allowed (sump, dispenser pan, interstice, etc.).

  • Go to the exact location (use Sensor ID + location label + your site map).
  • Verify physically: fuel present? source? active leak vs spill vs water intrusion carrying product.
  • Contain/clean safely per Petro Plus procedure.
  • Document: cause hypothesis + before/after photos + what was corrected.
Best practice: If it’s recurring at the same spot, treat it like a pattern (seal failure, conduit intrusion, fitting seep, repeat spill behavior).
Low Pressure Alarm (during dispensing)

Meaning: Low pump dispense pressure detected during a dispense; dispensing may halt if programmed.

  • Check if it’s one dispenser / one line or multiple.
  • Look for obvious: large line leak, empty tank, STP pressure issues, relay/contactor failure, power off.
  • Pair it with PLLD diagnostics (don’t chase symptoms without history).
Safety: If you suspect an active leak, treat it as a hazard. Make safe, escalate, document.
Gross line fail (3.0) / Line leak shutdown / Gross test fault

Meaning: Gross leak test failure or gross-test fault condition for a line (3.0 gph category).

  • Pull PLLD 3.0 diagnostics for that line (recent passes/fails + timestamps).
  • Confirm site conditions: dispenser activity can interrupt/affect test completion.
  • Verify hydraulics: STP performance, obvious leaks, dispenser symptoms.
  • Document results and escalate if shutdown/critical.
Periodic / Annual line fail (0.2 / 0.1)

Meaning: Precision test failure category (0.2 periodic / 0.1 annual).

  • Pull 0.2 / 0.1 diagnostics (don’t assume “leak” without reviewing invalid/rejected reasons).
  • Confirm the site can actually get no-dispense idle time needed to complete testing.
  • Document: which line, what the console reported, and what was verified physically.
Best practice: If a site can’t stay idle, you’ll see “bad tests” and confusion. Document the traffic reality.
Sensor Out Alarm

Meaning: Sensor setup incorrect, sensor disconnected, or not functioning properly.

  • Confirm the sensor exists and is the correct one for that location (site map).
  • Inspect wiring/conduit, water intrusion, damage, loose connections.
  • If likely setup/config: screenshot + escalate (don’t “random-program” in the field).
Setup Data Warning

Meaning: Device setup data problem / setup parameters need recheck.

  • Capture screenshots of the warning and the referenced device/tank/line.
  • Verify you are looking at the right device mapping (site map vs console labels).
  • Escalate with evidence if programming changes are needed.
High Water Alarm (water in tank)

Meaning: Water in the tank exceeds the programmed High Water Alarm limit.

  • Screenshot: water level/status screen and tank number.
  • Confirm it’s tied to the correct tank/probe and not a mismatch/mapping issue.
  • Follow Petro Plus water procedure (removal/containment/escalation) and document before/after.
Important: Treat tank water alarms as serious. Document clearly for the office and customer.
Diagnostics

PLLD / Line Tests (status + where to look)

Goal: get the history/diagnostics first, then decide whether to re-run tests or chase hardware.

TLS-450PLUS (touchscreen) — quick paths

  • Menu → Diagnostics → PLLD → 0.2 GPH TESTS
  • Menu → Diagnostics → PLLD → 3.0 GPH TESTS
Best practice: Screenshot the diagnostics page for the affected line (shows recent pass/fail history).

TLS-350 (keypad) — common diagnostic steps

  1. Press MODE for DIAG MODE
  2. FUNCTION to PRESSURE LINE LEAK DIAG
  3. STEP to 0.2 DIAG (or other test screens as needed)
  4. Press PRINT (if printer available)
When you re-run tests (best practices)
  • No-dispense time matters. Heavy traffic can cause repeated “bad/invalid” tests.
  • One line at a time: identify the impacted line, pull history, then act.
  • Document conditions: “site could not remain idle” is valuable troubleshooting context.
  • Don’t promise outcomes: report what the console shows + what you verified physically.
Start Line Pressure Test (TLS-350 style prompt you’ll see)

You may see prompts like “Start Line Pressure Test” then select the line and select the test type (commonly 3.0). Follow site/company procedure before starting tests during business hours.

Best Practice

Site Mapping (find the right tank/sensor fast)

Most “ATG confusion” is really a mapping/labeling problem. Fix the map, fix the calls.

Build a one-page Site Map (per store)

  • Tanks: Tank #, product, probe type, notes
  • Lines: Line # → which dispensers/grades (rough is fine initially)
  • Sensors: Sensor ID + physical location (STP sump, dispenser pan, tank interstice, spill bucket)
  • E-stops: location(s) + what they affect
  • Photos: 1 wide shot + inside each sump/pan you service

Fast “Locate the Sensor” method

  1. From the alarm: capture Sensor ID and the location label (if available).
  2. Use the Site Map to walk straight to that containment point.
  3. Verify physically (fuel/water/debris/damage/placement).
  4. Document before/after and what you changed.
  5. Recheck the console status.
Best practice: If the location label is wrong, don’t waste the call fixing it—note it, then correct later with authorization.
Minimum “evidence set” for ATG calls
  • Alarm screen photo
  • Detail/diagnostic screen photo (tank/line/sensor)
  • Photo of the physical location (sump/pan/tank area)
  • Close-up of what you found (water/product/damage)
  • After photo + “status recheck” photo
